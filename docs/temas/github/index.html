Introducción a Git y GitHub

Control de versiones y colaboración de proyectos!

¿Qué es Git?

¿Qué es Git?

  • Es un sistema de control de versiones distribuido.
  • Permite rastrear cambios en el código fuente.
  • Facilita la colaboración en proyectos de software.
  • Creado por Linus Torvalds en 2005.

¿Qué es GitHub?

¿Qué es GitHub?

  • Una plataforma basada en Git para colaborar en proyectos.
  • Ofrece un espacio en la nube para alojar repositorios.
  • Incluye herramientas adicionales como issues, pull requests y wikis.
  • Ideal para proyectos de código abierto y privados.

Comandos básicos de Git

Comandos básicos de Git

  • Inicializar un repositorio: 
    git init
  • Agregar cambios al área de preparación: 
    git add .

Comandos básicos de Git

  • Hacer un commit: 
    git commit -m "Mensaje del commit"
  • Ver el historial de cambios: 
    git log

Clonación de repositorios

Clonar un repositorio es descargar su contenido a tu máquina local. 

git clone URL_DEL_REPOSITORIO

Ejemplo:

git clone https://github.com/usuario/repo.git

Commits y push

Commits

Nos sirven para guardar los cambios en tu repositorio local:

  • 
  git add .
  git commit -m "Descripción de los cambios"
Push

Nos sirve para enviar los cambios al repositorio remoto (GitHub): 

  • git push origin main

Conclusión

¡Con Git y GitHub, puedes organizar tu código y colaborar fácilmente!
