Que son los Web Components

Los Web Components son un conjunto de tecnologías web que permiten crear elementos personalizados reutilizables con funcionalidad encapsulada.

Beneficios

  • Reutilización de código
  • Encapsulación de estilos y comportamiento
  • Compatibilidad con frameworks modernos
  • Estándar web nativo
  • Mejor rendimiento

Características principales

  • Custom Elements
  • Shadow DOM
  • HTML Templates
  • ES Modules

Custom Elements

Permiten definir elementos HTML personalizados con su propio comportamiento y propiedades. 

                
  class MiElemento extends HTMLElement {
    constructor() {
      super();
      // Inicialización
    }
    
    connectedCallback() {
      // Cuando el elemento se añade al DOM
    }
  }
  
  customElements.define('mi-elemento', MiElemento);

Shadow DOM

  • Conjunto de API’s de JavaScript para crear un árbol DOM “sombra”.
  • Crea un nuevo elemento encapsulado
  • Puede tener un estilo y scripts sin miedo de colisiones con otras partes del documento 
                
  class MiElemento extends HTMLElement {
    constructor() {
      super();
      // Inicialización
      const shadow = this.attachShadow({mode: 'open'});
      shadow.innerHTML = `
       < styles> < /styles>
       < div> < /div>
      `;
    }

  }

ES Modules

Permiten importar y exportar componentes de forma modular, facilitando su reutilización. 

              
// mi-componente.js
export class MiComponente extends HTMLElement {
  // Implementación
}

// Uso en otro archivo
import { MiComponente } from './mi-componente.js';
customElements.define('mi-componente', MiComponente);

Plantillas

Estructura de la plantilla de un componente

  • constructor()
    • Su objetivo es crear el componente y añadirle el shadow DOM
  • connectedCallback()
    • Siempre se ejecuta cuando el componente se añade al DOM
  • disconnectedCallback()
    • Siempre se ejecuta cuando el componente se elimina del DOM
  • attributeChangedCallback()
    • Se ejecuta cuando se modifica un atributo del componente
  • adoptedCallback()
    • Se ejecuta cuando el componente se añade a un nuevo documento 
              
// Definición del componente
class MiTarjeta extends HTMLElement {
  constructor() {
    super();
    const shadow = this.attachShadow({mode: 'open'});
    shadow.innerHTML = `
    < styles>  .tarjeta {
          border: 1px solid #ccc;
          border-radius: 8px;
          padding: 16px;
          margin: 10px;
          box-shadow: 0 2px 5px rgba(0,0,0,0.1);
        }
        h2 {
          color: #333;
          margin-top: 0;
        }< /styles>
    < div class="tarjeta">
    < h2>< slot name="titulo">Título por defecto
    < p>< slot>Contenido por defecto
    < /div>
    `;
  }
}

// Registro del componente
customElements.define('mi-tarjeta', MiTarjeta);

Compatibilidad con navegadores

Los Web Components son compatibles con todos los navegadores modernos. Para navegadores antiguos, se pueden utilizar polyfills.

Integración con frameworks

Los Web Components pueden ser utilizados junto con frameworks como React, Angular o Vue, o como alternativa a ellos.

Mejores prácticas

  • Usar nombres con guión para evitar conflictos
  • Extender siempre de HTMLElement
  • Implementar el ciclo de vida completo
  • Utilizar atributos para configuración
  • Documentar la API del componente
